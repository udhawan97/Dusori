Released July 22, 2026. Version 0.5.0 answers a question the review queue could not: not just what to study next, but when a topic is worth returning to. Scheduling stays something you create, never something Dusori decides for you.

[Open Dusori](https://udhawan97.github.io/Dusori/app/) · [View the GitHub release](https://github.com/udhawan97/Dusori/releases/tag/v0.5.0)

## Review on a schedule you set

- Each row in **Review next** now offers **Got it** and **Needs work**.
- **Got it** moves the topic up a fixed ladder: 1, then 3, 7, 14, 30, and 60 days.
- **Needs work** returns it to the first rung, one day out. A first review of either kind is one day out.
- A scheduled topic rests out of the queue until it is due, and due topics sort to the top, most overdue first.
- A topic you never mark reviewed keeps exactly the deterministic order it had before: active topics first, least recently updated first.
- Paused topics are unaffected by any schedule they carry — pausing is your decision, and a due date does not override it.

Marking a review appends a line to the topic's dated update file, so reviews show up in recent activity and the seven-day recap alongside everything else you did.

## Where the schedule lives

Each reviewed topic gains one small machine-owned file:

```json
{
  "schemaVersion": 1,
  "topicSlug": "topic-slug",
  "repetition": 2,
  "lastReviewedOn": "2026-07-21",
  "dueOn": "2026-07-28"
}
```

`Topics/<topic-slug>/review.json` is created by your first review action and never before. It is schema-validated and written under the same expected-hash guard as every other machine file: if another editor changed it first, Dusori re-reads the current schedule and applies your outcome on top of it rather than overwriting.

Dates are the device's local calendar day, so a one-day interval means the next local day. The stored `dueOn` is a promise, not a recalculation — if the ladder ever changes, schedules already made keep the dates you were shown.

Older Dusori builds ignore the file entirely. ZIP export and import carry it through untouched, and deleting it only forgets the schedule for that topic.

## Run from a clone on any desktop

- `npm run setup` downloads the repository-pinned pnpm, installs dependencies, and builds the app and companion.
- `npm start` builds if needed and opens Dusori.
- Both work the same way on macOS, Windows, and Linux, with Node.js 24 LTS as the only prerequisite.

## What this release does not do

Dusori still generates no calendar entry, notification, reminder, or background task, and does nothing while the app is closed. An overdue topic is a position in a list, never an alert. Key-based or general web search, Ollama transforms, and unattended work remain roadmap items.
