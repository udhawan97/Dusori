Released July 21, 2026. Version 0.4.0 turns Dusori from a primarily reading-and-progress surface into a fuller local workbench: write notes, find any saved idea, follow backlinks, inspect workspace drift, and choose the next review from the files you already own.

[Open Dusori](https://udhawan97.github.io/Dusori/app/) · [View the GitHub release](https://github.com/udhawan97/Dusori/releases/tag/v0.4.0)

## Write without taking custody

- Create a Markdown note inside the selected topic and open it directly in the editor.
- Edit existing files under `Notes/` with a plain Markdown textarea.
- Save through the same tracked-hash protocol used by roadmap changes.
- If another editor wrote first, keep that external file active and review Dusori's sibling proposal before accepting it.

Every accepted write stays ordinary Markdown and adds a dated update entry.

## Find and reconnect local knowledge

- Search Markdown and text locally from the inspector. Matching is case- and accent-insensitive, every query word must be present, and results include bounded snippets.
- Open a result at its exact workspace path; source titles come from the topic manifest when available.
- See which documents link to the current file from reverse wikilink edges.
- Run **Workspace health** to list unresolved wikilinks and source-manifest/file drift without rewriting or quarantining anything.

Search creates no stored index. Backlinks and health are projections of the same portable files and graph edges.

## Review from evidence, not invented urgency

Today now includes **Review next** and a seven-day recap. Active topics come before paused topics; within each group, the least recently updated topic comes first. The recap reads bounded recent entries from `Updates/YYYY/MM/`.

Dusori does not invent a due date, calendar, spaced-repetition interval, or closed-app job. The queue and recap are reproducible from the current roadmap, state, and update files.

## Safer replacement imports

ZIP archives are validated in memory before Dusori asks to replace a browser workspace. The confirmation names the incoming workspace and reports topic and file counts. Invalid archives do not touch the destination; if a storage write fails during replacement, Dusori restores the previous snapshot.

Archives remain capped at 64 MiB compressed/expanded, 5,000 files, normalized workspace paths, and valid required schemas.

## Run the companion from npm

Node.js 24 users can launch the exact published package without cloning the repository:

```sh
npx dusori@0.4.0 --root /path/to/Dusori
```

`npx dusori@0.4.0 --help` and `--version` exit without starting a server. The package is built with the app shell and service worker, checked for version alignment, and smoke-tested from its packed tarball before publication.

The runtime boundary is unchanged: loopback only, a new token per run, an optional explicit root, and a lifetime tied to the terminal process. No account, telemetry, hosted application backend, AI model, general web search, generated schedule, or background daemon was added.
