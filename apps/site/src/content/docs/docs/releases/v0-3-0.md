Released July 21, 2026. Version 0.3.0 adds an optional local companion for capabilities the static browser app cannot safely provide alone, while keeping every remote request explicit and every workspace write local and reviewable.

[Open Dusori](https://udhawan97.github.io/Dusori/app/) · [View the GitHub release](https://github.com/udhawan97/Dusori/releases/tag/v0.3.0)

## Readable pages, only when you ask

- Open Dusori through the companion and a saved URL source gains **Fetch full content**.
- Review an exact-host disclosure before the first byte leaves your machine.
- Preview the extracted Markdown before it replaces the reference stub.
- Keep an external edit active if the source changed since Dusori last read it; the replacement is refused as a conflict rather than written silently.
- Preserve the source's URL identity, provenance, graph position, and dated update history.

The companion checks the destination and every redirect against private, reserved, and other non-public address ranges, follows at most three redirects, accepts only HTML and plain text, caps the response at 4 MiB, and times out after 15 seconds. Readable output remains subject to Dusori's 2 MiB source limit.

## Better Microsoft Learn ranking

When connected, the Research panel uses Microsoft's ranked Learn search through the companion. If that request fails, Dusori falls back to the keyless catalog ranking already available in the hosted app. English Wikipedia research remains consent-gated and browser-based.

## Session and keyboard trust

- The launch token and companion origin are consumed into memory and removed from the address and current history entry immediately after launch, while normal topic and view parameters remain intact.
- A loopback endpoint is shown as connected only when it returns the versioned Dusori companion health contract. Static HTML fallbacks and wrong-service JSON stay disconnected.
- Source confirmation, source preview, research consent, research preview, and Obsidian guidance now use true browser modals. Tab and Shift+Tab stay inside; Escape closes once; focus returns to the exact invoking control.

## Compatibility warning for v0.2.0

Do not reopen a workspace containing a companion-upgraded source with v0.2.0. That reader can reject the newer `companion` provenance value and rename `Sources/manifest.json` to `manifest.json.invalid-<timestamp>`. Source content is never deleted, but the library and ZIP export remain incomplete until the manifest is restored.

Update to v0.3.0 first. If the rename already happened, update Dusori and rename the `.invalid-<timestamp>` manifest back to `manifest.json`.

## Install boundary

The GitHub release publishes the hosted app and companion source. It does **not** publish the `dusori` package to npm. Build the companion from a clone with Node.js 24 and pnpm 11; `npx dusori` was the intended command at release time. npm later reserved that unscoped name under its similarity policy, so v0.4.0 publishes the companion as [`@udhawan97/dusori`](../v0-4-0/).

No account, telemetry, hosted application backend, paid API key, AI model, general-purpose web search, or always-running daemon is required or added.
